# Resolve Top-Down Ordnance Through Impact Columns

Mortars and orbital bombardments will resolve their target as a same-X/Y impact column across connected Z-levels, because top-down ordnance should hit the highest reachable blocking surface instead of phasing through upper floors to the originally selected level. Existing area-info permission checks remain the authority for whether a surface may be affected: if an area blocks mortar fire or orbital bombardment anywhere the ordnance would strike, the column is blocked. Mortar HE is carving ordnance: it explodes at the first allowed surface, waits for tile damage to create an opening, then continues down only through that opening to the next allowed blocking surface. Orbital bombardments use the lowest allowed non-space surface as their terminal impact and create smaller transit blasts on intermediate surfaces they pass through. Rangefinder and tactical area info should report these top-down permissions so targeting feedback matches the actual impact behavior.
